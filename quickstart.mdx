## Prerequisites

<Callout kind="info" title="What you'll need">
- A Deepvue account and API key from the [API Dashboard](https://dashboard.deepvue.tech/)
- Postman (optional, for no-code testing) or your preferred coding environment
- Base URL: `https://production.deepvue.tech`
</Callout>

Deepvue's REST APIs use standard HTTP methods and JSON responses. Authentication requires a Bearer token in the `Authorization` header. Follow these steps to make your first API call to retrieve a list of banking accounts.

## Step-by-Step Quickstart

<Steps>
  <Step title="Create Account and Get API Key" icon="monitor">
    Visit the [Deepvue Dashboard](https://dashboard.deepvue.tech/) and sign up or log in.
    
    Navigate to the API section to generate your API key. Copy it securely—it looks like `dv_sk_abc123def456`.

    <Callout kind="tip">
      Store your API key as an environment variable: `DEEPVUE_API_KEY=your-key-here`.
    </Callout>
  </Step>

  <Step title="Download Postman Collection" icon="download">
    Import the official [Deepvue Postman Collection](https://hi.deepvue.tech/postman) into Postman.
    
    Set up a Postman environment:
    
````json
{
  "base_url": "https://production.deepvue.tech",
  "api_key": "YOUR_API_KEY"
}
````
    
    This prepares you for testing without writing code.
  </Step>

  <Step title="Execute Sample Request" icon="play">
    Use the sample request below to fetch banking accounts. Replace `YOUR_API_KEY` with your actual key.
    
    In Postman, select the collection, set the Authorization header to `Bearer {{api_key}}`, and hit Send.
  </Step>

  <Step title="Verify Response" icon="check-circle">
    A successful response returns `200 OK` with JSON data like:
    
````json
{
  "success": true,
  "accounts": [
    {
      "id": "acc_123",
      "balance": 1500.50,
      "currency": "USD"
    }
  ]
}
````
    
    Errors show standard HTTP codes (e.g., `401 Unauthorized` if key is invalid).
  </Step>
</Steps>

## Sample API Request

Make a GET request to list your banking accounts. All requests require the `Authorization` header.

<Request
  tabs="cURL,Python,Node.js"
  default-tab="1"
  show-lines="true"
>
  ```bash
curl -X GET 'https://production.deepvue.tech/api/v1/banking/accounts' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json'
  ```

  ```python
import requests
import os

url = "https://production.deepvue.tech/api/v1/banking/accounts"
headers = {
    "Authorization": f"Bearer {os.getenv('DEEPVUE_API_KEY')}",
    "Content-Type": "application/json"
}

response = requests.get(url, headers=headers)
print(response.json())
  ```

  ```javascript
const fetch = require('node-fetch');
const apiKey = process.env.DEEPVUE_API_KEY;

const response = await fetch('https://production.deepvue.tech/api/v1/banking/accounts', {
  method: 'GET',
  headers: {
    'Authorization': `Bearer ${apiKey}`,
    'Content-Type': 'application/json'
  }
});

const data = await response.json();
console.log(data);
  ```
</Request>

## Troubleshooting Common Issues

<Expandable title="API Key Errors (401)" default-open="true">
Check that your `Authorization` header is exactly `Bearer YOUR_API_KEY` (no extra spaces). Regenerate the key in the dashboard if needed.
</Expandable>

<Expandable title="Rate Limits (429)">
Deepvue enforces rate limits. Implement exponential backoff in production code.
</Expandable>
